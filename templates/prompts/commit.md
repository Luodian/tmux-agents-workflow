# Commit message style

Format: `<scope>: <imperative subject>` (50 chars or less).

Body (optional, wrap at 72): explain *why* the change exists. The diff shows what.

Examples:
- `auth: add token refresh on 401`
- `tests: extend coverage for retry path`
- `docs: clarify worktree gate trigger`

Avoid: "Update X", "Fix bug" — empty subjects.
