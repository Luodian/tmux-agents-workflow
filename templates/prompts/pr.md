# PR description — repo style

When opening a PR for this repo, use this structure:

## Summary
1–3 plain-English bullets. Lead with the *why*, not the *what*.

## Changes
- `path/to/file.py`: what changed and why
- `…`

## Test plan
- [ ] Manual check: …
- [ ] CI:

## Notes
Followups, known limitations, rollback plan if any.
